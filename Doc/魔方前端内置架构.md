# 魔方前端内置架构

## 1. 架构概览

### 终态架构

```
用户 ASP.NET 项目
├── 引用 NewLife.Cube (API核心)            → 控制器/实体/权限/OAuth/服务
├── 引用 NewLife.Cube.Vue (前端皮肤)       → Vue3 + Element Plus 编译产物嵌入
│   或 NewLife.Cube.React (前端皮肤)       → React + Ant Design 编译产物嵌入
└── dotnet run → 浏览器打开即有完整管理后台
```

### 嵌入式 SPA 架构

```
┌─────────────────────────────────────────────────────────────┐
│                    ASP.NET Core 主机                         │
├─────────────────────────────────────────────────────────────┤
│  中间件管道                                                  │
│  ┌─────────────┐  ┌──────────────┐  ┌───────────────────┐   │
│  │ 认证/授权    │→│ 静态文件      │→│ 路由 + API 端点    │   │
│  │ JWT Bearer  │  │ Embedded +   │  │ EntityController   │   │
│  │             │  │ Physical     │  │ GetFields/CRUD     │   │
│  └─────────────┘  └──────────────┘  └───────────────────┘   │
│                          ↑                                   │
│              CubeEmbeddedFileProvider                        │
│              (程序集嵌入资源 → HTTP 响应)                      │
├──────────────┬──────────────────────────────────────────────┤
│  Cube.Vue    │  wwwroot/ (嵌入)                              │
│  程序集       │  ├── index.html                              │
│              │  ├── assets/                                  │
│              │  │   ├── index-xxx.js  (Vue 编译产物)         │
│              │  │   └── index-xxx.css                       │
│              │  └── favicon.ico                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 与 MVC 皮肤的对比

| 维度 | MVC 皮肤（AdminLTE/Tabler） | 前端皮肤（Vue/React） |
|------|--------------------------|---------------------|
| SDK | Microsoft.NET.Sdk.Web | Microsoft.NET.Sdk |
| 依赖 | NewLife.CubeNC (MVC) | NewLife.Cube (API) |
| 输出目录 | Bin/ | Bin3/ |
| 包含内容 | Razor Views + wwwroot | wwwroot（编译产物） |
| 渲染方式 | 服务端 Razor | 客户端 SPA |
| 注册方式 | UIService.AddTheme/Skin | 静态文件 + SPA 回退 |
| 前端源码 | 无（直接 CSS/JS） | web/ 目录（需编译） |

## 2. 数据模型

本项目不新增数据实体，复用现有魔方实体体系：

| 实体 | 用途 | 所在项目 |
|------|------|---------|
| User / Role / Menu | 用户权限菜单 | NewLife.Cube (Entity/) |
| App / AppLog | 应用系统与日志 | NewLife.Cube (Entity/) |
| 参数字典 / 地区 | 配置与字典 | NewLife.Cube (Entity/) |

前端通过 `GetFields` API 获取字段元数据，动态渲染 UI，不依赖特定实体结构。

## 3. 接口设计

### 3.1 已有核心接口（无需修改）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 列表查询 | GET | `/{area}/{controller}` | 返回 `ApiListResponse<T>` |
| 查看详情 | GET | `/{area}/{controller}/Detail?id=` | 返回 `ApiResponse<T>` |
| 字段元数据 | GET | `/{area}/{controller}/GetFields?kind=` | 返回 `List<DataField>` |
| 新增 | POST | `/{area}/{controller}` | 返回 `ApiResponse<T>` |
| 编辑 | PUT | `/{area}/{controller}` | 返回 `ApiResponse<T>` |
| 删除 | DELETE | `/{area}/{controller}?id=` | 返回 `ApiResponse` |

### 3.2 需补充的接口

| 接口 | 方法 | 路径 | 入参 | 出参 | 说明 |
|------|------|------|------|------|------|
| 菜单树 | GET | `/Admin/Menu/GetTree` | 无 | `List<MenuTree>` | 前端动态路由必需 |
| 当前用户 | GET | `/Admin/User/Info` | 无 | `UserInfo` | 用户名/头像/角色/权限 |
| 登录配置 | GET | `/Admin/User/GetLoginConfig` | 无 | `LoginConfigModel` | OAuth 提供商列表、注册策略 |
| 字典查询 | GET | `/Admin/Dict/Lookup?codes=` | codes | `Dictionary<String,List>` | 枚举/下拉选项数据源 |
| 系统配置 | GET/PUT | `/Cube/Setting` | 无/配置对象 | `CubeSetting` | 读写系统配置 |

### 3.3 GetFields 元数据增强

现有 `DataField` 需确保返回以下字段供前端使用：

| 字段 | 类型 | 说明 | 前端用途 |
|------|------|------|---------|
| Name | String | 字段代码名 | 数据绑定 key |
| DisplayName | String | 显示名称 | 表头/标签 |
| TypeName | String | .NET 类型名 | 组件类型推导 |
| ItemType | String | UI 控件类型 | 直接映射前端组件 |
| Width | String | 列宽 | 表格列宽 |
| DataSource | String | 数据源标识 | 下拉选项来源 |
| Required | Boolean | 是否必填 | 表单验证 |
| ReadOnly | Boolean | 是否只读 | 表单控制 |
| Multiple | Boolean | 是否多选 | 搜索字段 |
| Url | String | 链接模板 | 列表点击跳转 |

## 4. 技术选型

| 领域 | 选型 | 理由 |
|------|------|------|
| Vue 前端基础 | QuickVue (V3 + Element Plus + Pinia) | 最成熟，usePage hook 零代码 CRUD，生产级验证 |
| React 前端基础 | CubeAntd (React 18 + Ant Design 5 + UMI Max) | Pro Components 企业级，TypeScript 严格 |
| 静态文件嵌入 | CubeEmbeddedFileProvider | 已有成熟实现，6 个 MVC 皮肤验证 |
| 模块自动发现 | IModule 接口 | 已有模式，无需额外框架 |
| 前端构建 | Vite (Vue) / UMI (React) | 各自项目原有工具链 |
| 包管理 | pnpm | 快速、磁盘友好 |

## 5. 关键设计决策

| 决策点 | 方案 | 备选方案 | 选择理由 |
|--------|------|---------|---------|
| 前端引入方式 | 代码迁移到当前库 | Git 子模块 | 与现有皮肤包模式一致；版本同步；CI 更简单 |
| 项目 SDK | Microsoft.NET.Sdk + FrameworkReference | Microsoft.NET.Sdk.Web | API 版用 plain SDK，保持一致；无 Razor 不需要 Web SDK |
| 输出目录 | Bin3/ | Bin/ | 与 API 核心同目录，NuGet 打包一致 |
| 编译产物 | CI 编译，不提交 Git | 提交到 Git | 仓库干净；前端更新频繁避免 diff 噪音 |
| SPA 回退 | 自定义中间件 | MapFallbackToFile | 嵌入资源不在物理路径，需自定义 |
| 原仓库 | 保留独立维护 | 归档 | 社区可能直接使用独立前端 |

### 6.2 实体管理控制器 —— 已完整覆盖 ✅

经详细核查，API 版 Admin 和 Cube 区域已拥有 **30+ 个控制器**，与 MVC 版完全对等：

| 区域 | 控制器数 | 代表性控制器 |
|------|---------|------------|
| Admin | 20+ | UserController, RoleController, MenuController, DepartmentController, LogController, ParameterController, OAuthConfigController, UserTokenController, UserConnectController, TenantController 等 |
| Cube | 9 | AppController, CronJobController, AttachmentController, AreaController, ModelTableController 等 |

基类模式分布：
- `EntityController<TEntity, TModel>`：23 个（完整 CRUD + DTO 映射）
- `ReadOnlyEntityController<TEntity>`：4 个（AppLog, XLog, OAuthLog, UserStat）
- `EntityTreeController<TEntity, TModel>`：1 个（Menu）
- `ConfigController<TConfig>`：4 个（CubeSetting, Setting, SysConfig, XCodeSetting）
- `EntityController<TEntity>`：2 个（ModelTable, ModelColumn，实体即 DTO）

**结论**：API 版无控制器缺口，前端可直接全量对接。

### 6.3 皮肤注册方式

经验证，MVC 皮肤（AdminLTE/Tabler 等）均通过**显式调用**注册，非 IModule 自动发现：

```csharp
// CubeDemoNC/Startup.cs
app.UseCube(env);
app.UseAdminLTE(env);   // 显式调用
app.UseTabler(env);     // 显式调用
```

Vue/React 皮肤沿用同模式：

```csharp
// CubeDemo/Program.cs（API 版）
app.UseCube(builder.Environment);
app.UseVue(builder.Environment);    // 一行即启用 Vue 前端
// 或
app.UseReact(builder.Environment);  // 一行即启用 React 前端
```

IModule 机制作为辅助路径保留，通过 AppModule 管理页可注册到数据库实现免代码启用。

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| NuGet 包体积膨胀 | 下载慢，磁盘占用大 | Vite 压缩 + gzip + 拆分 vendor chunk；评估 CDN 可选方案 |
| GetFields 元数据不足 | 前端渲染缺失字段 | 迭代 2 逐字段对齐 MVC 版，参照 QuickVue/CubeAntd 已对接的字段 |
| 前端更新频率高于后端 | NuGet 版本耦合 | 皮肤包使用独立版本号（1.x），可独立发布 |
| CubeEmbeddedFileProvider 跨项目兼容 | 编译可能失败 | T001 优先验证，确保 API SDK 模式下可用 |
| CI 需 Node.js 环境 | 编译复杂度增加 | GitHub Actions 预装 Node.js；提供 build.ps1 本地脚本 |
| SPA 回退与 API 路由冲突 | 页面白屏或 API 404 | Accept header 检测 + 路径排除规则 |
