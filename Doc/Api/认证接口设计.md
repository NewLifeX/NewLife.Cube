## 2. 后端接口规范

### 2.1 获取登录配置

> **注意**：原 `/Cube/SiteInfo` 接口已移除，登录页所需的展示信息（`name`/`copyright`/`logo` 等）均包含在 `/Auth/LoginConfig` 响应中。

```
GET /Auth/LoginConfig?tenant={tenant}
```

`tenant` 可选，支持租户 ID（数字）、租户 Code 或租户名称；为空时返回全局站点信息。

**响应示例（部分字段）**

```json
{
  "code": 0,
  "data": {
    "name": "魔方WebApi",
    "copyright": "©2002-2026 NewLife Powered by .NET 9.0.0",
    "registration": "粤ICP备16014330号-1",
    "loginTip": "",
    "logo": "",
    "loginLogo": "",
    "loginBackground": ""
  }
}
```

**说明**
- `copyright`：服务端已替换 `{now:yyyy}` 等变量，前端直接显示（Vue 皮肤用 `v-html` 渲染，支持 HTML 链接）
- `loginLogo` / `loginBackground`：为空时前端降级到皮肤内置静态资源
- 传入 `tenant` 时，`name` 优先返回租户名称，其余字段仍读全局配置

**完整响应示例（含嵌套开关字段）**

```json
{
  "code": 0,
  "data": {
    "name": "魔方WebApi",
    "loginTip": "",
    "loginLogo": "",
    "loginBackground": "",
    "login": {
      "password": true,
      "sms": false,
      "mail": false,
      "captcha": false
    },
    "register": {
      "enabled": true,
      "password": true,
      "sms": false,
      "mail": false,
      "captcha": false
    },
    "oauth": [
      {
        "name": "NewLife",
        "logo": "/Content/images/logo/NewLife.png",
        "nickName": "新生命用户中心"
      }
    ],
    "security": {
      "challengeRequired": false,
      "mfaAvailable": false
    }
  }
}
```

**说明**
- `login.captcha` 为 `true` 时，前端需先调用 `GET /Auth/Captcha` 获取图片验证码，登录请求中附带 `captchaId` + `captchaCode`
- `security.mfaAvailable` 为 `true` 时，已开启 MFA 的用户登录后会收到 `mfa_required:xxx` 消息，需进行二步验证
- `oauth` 仅返回 `Visible=true` 的 OAuth 提供商


### 2.3 获取图片验证码

```
GET /Auth/Captcha
```

**响应示例**

```json
{
  "code": 0,
  "data": {
    "captchaId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "image": "<svg xmlns=\"http://www.w3.org/2000/svg\">...</svg>"
  }
}
```

**使用流程**
1. 前端调用本接口获取 `captchaId` 和 SVG 图片
2. 将 SVG 作为 `<img src="data:image/svg+xml;base64,...">` 或直接 `innerHTML` 显示
3. 用户填写答案后，在 `login` / `register` / `sendCode` 请求中附带 `captchaId` + `captchaCode`
4. 验证通过后验证码立即失效（防重放）；验证失败需重新获取


### 2.4 密码登录

```
POST /Auth/Login
Content-Type: application/json

{
  "username": "admin",
  "password": "...",
  "challengeId": "",
  "captchaId": "",
  "captchaCode": ""
}
```

| 字段 | 说明 |
|------|------|
| `password` | 明文密码，或 Challenge-Response 流程加密后的 Base64 密文 |
| `challengeId` | Challenge 挑战标识，非空时后端解密 `password`；不走加密流程时留空 |
| `captchaId` | 图片验证码 ID，`login.captcha=true` 时必填 |
| `captchaCode` | 图片验证码答案，`login.captcha=true` 时必填 |

**普通登录响应：**

```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "accessToken": "...",
    "refreshToken": "...",
    "expireIn": 7200
  }
}
```

**MFA 触发响应（code=0，data 为 null，message 含 mfaToken）：**

```json
{
  "code": 0,
  "message": "mfa_required:eyJhbGciOiJub25lIn0...",
  "data": null
}
```

前端检测 `message` 是否以 `mfa_required:` 开头，是则提取 mfaToken 进入二步验证流程。

```
GET /Auth/Challenge
```

**响应**

```json
{
  "code": 0,
  "data": {
    "challengeId": "a3f2c8e1-4b7d-11ef-9012-0800200c9a66",
    "publicKey": "-----BEGIN PUBLIC KEY-----\nMIIBIjAN...\n-----END PUBLIC KEY-----"
  }
}
```

前端用 `publicKey`（PKCS#8 SPKI 格式）以 **RSA-OAEP/SHA-256** 加密密码后，将密文和 `challengeId` 一起提交给 `/Auth/Login`。

**安全说明**
- `challengeId` 有效期 300 秒，使用一次后立即失效，防止重放攻击
- 若加密失败（浏览器不支持 Web Crypto API），可降级为明文，此时 `challengeId` 不传，但需要服务端 `AllowPlainPassword = true`

### 2.5 发送验证码

```
POST /Auth/SendCode
Content-Type: application/json

{
  "channel": "Sms",
  "username": "13800138000",
  "action": "Login"
}
```

**注意**：`channel` 值区分大小写，手机为 `Sms`，邮箱为 `Mail`。  
`action` 可选值：`Login`（登录）、`Register`（注册）、`ResetPassword`（重置密码）。

**响应**

```json
{
  "code": 0,
  "data": 123456789
}
```

### 2.6 验证码登录

验证码登录与密码登录共用 `/Auth/Login` 接口，通过 `category` 字段区分登录方式：

```
POST /Auth/Login
Content-Type: application/json

{
  "username": "13800138000",
  "password": "1234",
  "category": "mobile"
}
```

`category` 取值：`""` = 密码登录（默认）、`"mobile"` = 手机验证码、`"mail"` = 邮箱验证码、`"oauth"` = 第三方登录。响应格式同 2.4。

### 2.7 忘记密码——重置密码

```
POST /Auth/ResetPassword
Content-Type: application/json

{
  "username": "13800138000",
  "code": "1234",
  "newPassword": "NewPass@2026",
  "confirmPassword": "NewPass@2026",
  "challengeId": ""
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `username` | 是 | 手机号或邮箱（与发送验证码时一致） |
| `code` | 是 | 短信/邮件验证码 |
| `newPassword` | 是 | 新密码（明文或加密密文） |
| `confirmPassword` | 是 | 确认密码（明文或加密密文） |
| `challengeId` | 否 | Challenge 挑战标识，用于解密新密码 |

调用前须先调用 `POST /Auth/SendCode` 并指定 `action: "ResetPassword"` 发送验证码。
若需要加密传输，还需先调用 `GET /Auth/Challenge` 获取挑战码。

**响应**

```json
{
  "code": 0,
  "message": "密码重置成功",
  "data": true
}
```

### 2.8 注册

SPA 皮肤（Vue/React 等）：前端跳转到独立 `/register` 路由页面。  
MVC 皮肤（Metronic8 等）：`POST /Admin/User/Register`，Bootstrap Tab 内表单提交。

## 3. 配置项说明（CubeSetting）

### 界面配置

| 配置项 | 说明 | 默认值 |
|--------|------|-------|
| `LoginTip` | 登录页提示信息，留空不显示 | 空 |
| `LoginLogo` | 登录页左上角 Logo 图片路径，留空使用皮肤内置 | 空 |
| `LoginBackground` | 登录页左侧背景大图路径，留空使用皮肤内置 | 空 |
| `Copyright` | 版权文字，支持 `{now:yyyy}`/`{runtime}`/`{framework}` 变量 | 空（不显示） |
| `Registration` | 备案号，留空不显示 | `沪ICP备10000000号` |

### 用户登录

| 配置项 | 说明 | 默认值 |
|--------|------|-------|
| `AllowLogin` | 允许密码登录 | `true` |
| `AllowRegister` | 允许注册 | `true` |
| `EnableSms` | 启用短信验证码登录（同时控制忘记密码短信通道） | `false` |
| `EnableMail` | 启用邮箱验证码登录（同时控制忘记密码邮件通道） | `false` |
| `AllowPlainPassword` | 允许明文密码传输。设为 `false` 后强制要求前端通过 Challenge-Response 加密密码 | `true` |

> 忘记密码入口依赖通知通道：`EnableSms` 或 `EnableMail` 至少有一个为 `true` 才会在登录页显示"忘记密码"链接。

## 4. 前端实现规范

### 4.1 SPA 皮肤（Vue / ArcoVue / NaiveUI / TDesign / React / Shadcn）

#### 登录页结构

```
/login
  ├── 左侧装饰区
  │     ├── Logo（loginLogo 或皮肤内置）
  │     └── 背景图（loginBackground 或皮肤内置）
  └── 右侧表单区
        ├── 系统名称
        ├── Tab 标签页
        │     ├── 密码登录（allowLogin=true 显示）
        │     ├── 手机验证码（enableSms=true 显示）
        │     └── 邮箱验证码（enableMail=true 显示）
        ├── 忘记密码链接（enableSms||enableMail=true 显示）
        ├── OAuth 第三方登录（providers 非空时显示，分割线 + 图标列表）
        ├── 注册入口（allowRegister=true 显示）
        └── 底部版权信息（copyright / registration）

/forgot-password
  ├── Step 1：输入手机号或邮箱，选择接收方式，发送验证码
  └── Step 2：输入验证码 + 新密码 + 确认密码，提交重置
        └── 成功后跳转到 /login

/register
  └── 注册表单（用户名、邮箱、密码、确认密码）
```

#### Challenge-Response 加密流程（所有皮肤通用，由 `@cube/auth-logic` 封装）

```typescript
// packages/auth-logic/src/index.ts（login 函数节选）
async function login(username: string, password: string) {
  let payload = { username, password, challengeId: '' };
  try {
    const cr = await api.get('/Auth/Challenge');
    if (cr.data?.publicKey) {
      const encrypted = await encryptPassword(password, cr.data.publicKey);
      if (encrypted) {
        payload.password = encrypted;
        payload.challengeId = cr.data.challengeId;
      }
    }
  } catch {
    // 降级：Challenge 失败时继续使用明文（需服务端 AllowPlainPassword=true）
  }
  return api.post('/Auth/Login', payload);
}
```

#### category 字段规范

所有登录/注册请求中的 `category` 字段统一使用字符串：

| 值 | 说明 |
|----|------|
| `""` | 密码登录（默认，可省略） |
| `"mobile"` | 手机验证码登录/注册 |
| `"mail"` | 邮箱验证码登录/注册 |
| `"oauth"` | 第三方 OAuth 登录 |

后端对应枚举为 `AuthCategory { Password=0, Mobile=1, Mail=2, OAuth=3 }`。

#### OAuth 按钮渲染细节

- 图标：`<img :src="item.logo" />`，`item.logo` 由后端返回绝对路径，无需前端处理
- 文字：优先 `item.nickName`，降级到 `item.name`
- 登录跳转：`window.location.href = '/Sso/Login/' + item.name + '?r=' + encodeURIComponent(returnUrl)`（需全页刷新完成 OAuth 流）
- `returnUrl` 默认为登录后拟访问的页面，若无则为首页

#### 版权渲染细节

- `copyright` 字段已由服务端替换变量（含 HTML 链接），前端使用 `v-html`（Vue）或 `dangerouslySetInnerHTML`（React）渲染
- `registration` 字段为纯文本，包裹在 `<a href="https://www.beianx.cn/">` 链接内

#### 验证码发送逻辑

1. 点击"发送验证码"按钮，调用 `POST /Auth/SendCode`
2. 成功后开始 60 秒倒计时，按钮禁用并显示剩余秒数
3. 倒计时结束后按钮恢复可点击状态
4. 失败时显示错误信息，按钮保持可用

### 4.2 MVC 皮肤（Metronic8 / Metronic v3 / ElementUI / LayuiAdmin）

- Tab 结构使用 Bootstrap 的 `data-toggle="tab"` 切换
- 验证码发送使用 jQuery AJAX 调用 `/Admin/User/SendVerifyCode`（原有 MVC 端点）
- 注册使用 Bootstrap Tab 切换到本页内的注册面板（`#Register`）
- 忘记密码使用 Bootstrap Tab 切换到本页内的忘记密码面板（`#ForgotPassword`），两步骤通过 JS 控制显示/隐藏
- 版权信息调用 `config.GetCopyright()` 方法渲染

#### MVC 皮肤忘记密码面板流程

```
点击"忘记密码"链接
  └── Bootstrap Tab 切换到 #ForgotPassword 面板（Step 1）

Step 1：输入手机号或邮箱，点击发送验证码
  └── AJAX POST /Auth/SendCode { channel, username, action: "ResetPassword" }
        └── 成功 → 显示 Step 2（输入验证码和新密码）

Step 2：输入验证码 + 新密码 + 确认密码，点击提交
  └── AJAX POST /Auth/ResetPassword { username, code, newPassword, confirmPassword }
        ├── 成功 → 切换回登录 Tab，显示"密码重置成功，请登录"提示
        └── 失败 → 显示错误信息在当前 Step 2
```

#### MVC 皮肤 Challenge-Response 实现

MVC 皮肤登录表单提交时，通过内联 JS 函数拦截 `onsubmit` 事件完成加密。

```javascript
// 表单结构（含 challengeId 隐藏字段）
<form id="login_form" onsubmit="return handleLoginSubmit(event)">
  <input type="hidden" name="challengeId" id="challenge_id_input" value="" />
  <input type="password" name="password" id="password_field" />
</form>

// 加密并提交
async function handleLoginSubmit(e) {
  e.preventDefault();
  try {
    const resp = await fetch('/Auth/Challenge');
    const { data } = await resp.json();
    if (data?.publicKey) {
      const encrypted = await encryptRSA(document.getElementById('password_field').value, data.publicKey);
      if (encrypted) {
        document.getElementById('password_field').value = encrypted;
        document.getElementById('challenge_id_input').value = data.challengeId;
      }
    }
  } catch { /* 降级明文 */ }
  document.getElementById('login_form').submit();
  return false;
}
```

## 5. 数据流图

```
浏览器加载登录页
    │
    └── GET /Auth/LoginConfig → 展示版权、更新 document.title、获取展示信息（loginBackground/loginLogo/copyright 等）、控制 Tab 显示/隐藏、渲染 OAuth 提供者列表

用户点击密码登录
    ├── GET /Auth/Challenge → 获取 challengeId + publicKey
    ├── 【浏览器端】RSA-OAEP/SHA-256 加密密码
    └── POST /Auth/Login { username, password: [密文], challengeId } → 成功后跳转首页

用户点击手机验证码登录
    ├── POST /Auth/SendCode { channel: "Sms", username, action: "Login" }
    └── POST /Auth/Login { username, password: 验证码, category: "mobile" }

用户点击邮箱验证码登录
    ├── POST /Auth/SendCode { channel: "Mail", username, action: "Login" }
    └── POST /Auth/Login { username, password: 验证码, category: "mail" }

用户点击第三方登录
    └── Location: /Sso/Login/{name}[?r=returnUrl] → OAuth 流程由后端处理

用户点击注册
    └── SPA: router.push('/register')
        MVC: Bootstrap Tab 切换到 #Register 面板

用户点击忘记密码
    ├── SPA: router.push('/forgot-password')
    │     ├── Step 1: POST /Auth/SendCode { channel, username, action: "ResetPassword" }
    │     └── Step 2: GET /Auth/Challenge 获取挑战码 → 加密新密码 → POST /Auth/ResetPassword { username, code, newPassword, confirmPassword, challengeId }
    │                    └── 成功 → router.push('/login')
    └── MVC: Bootstrap Tab 切换到 #ForgotPassword 面板（同上两步，通过 AJAX 完成）
```

## 6. 接口兼容性说明

| 接口 | 路径 | 状态 |
|------|------|------|
| 登录页配置（包含站点信息） | `GET /Auth/LoginConfig[?tenant=]` | **当前版本**，支持租户参数 |
| 图片验证码 | `GET /Auth/Captcha` | **当前版本** |
| 站点信息 | `GET /Cube/SiteInfo` | **已移除**，请改用 `GET /Auth/LoginConfig` |
| Challenge 公钥 | `GET /Auth/Challenge` | **当前版本** |
| 重置密码 | `POST /Auth/ResetPassword` | **当前版本** |
| 密码登录 | `POST /Auth/Login` | **当前版本**，新增 `captchaId`/`captchaCode` 字段 |
| 验证码登录 | `POST /Auth/Login`（`category: "mobile"/"mail"`） | **当前版本**，已与密码登录合并，旧 `/Auth/LoginByCode` 已移除 |
| 发送验证码 | `POST /Auth/SendCode` | **当前版本**，新增 `captchaId`/`captchaCode` 字段（防短信轰炸） |
| 注册 | `POST /Auth/Register` | **当前版本**，新增 `captchaId`/`captchaCode` 字段 |
| 刷新令牌 | `POST /Auth/Refresh` | **当前版本**，令牌轮换（旧 refreshToken 使用后立即失效） |
| MFA 二步验证 | `POST /Mfa/Verify` | **新增** |
| MFA 初始化 | `GET /Mfa/Setup` | **新增**，需登录 |
| MFA 激活 | `POST /Mfa/Activate` | **新增**，需登录 |
| MFA 禁用 | `POST /Mfa/Disable` | **新增**，需登录 |
| MFA 状态 | `GET /Mfa/Status` | **新增**，需登录 |
| MVC 登录（保留） | `/Admin/User/Login` | **保留**，供 MVC 皮肤和 SSO 回调使用 |

- `/Auth/*` 系列接口由 `AuthController` 提供，适配 SPA 前端、MVC 皮肤的 AJAX 调用
- `/Mfa/*` 接口由独立的 `MfaController` 提供
- `/Admin/User/*` 原有 MVC 接口保留，供 MVC 皮肤表单直传和旧版项目使用
- 两套接口共享 `UserService` 业务层，行为一致

## 7. 图片验证码集成指南

### CaptchaScene 位掩码

| 位 | 值 | 含义 |
|----|-----|------|
| 0 | 0  | 不启用图片验证码 |
| 1 | 1  | 登录时需要验证码 |
| 2 | 2  | 注册时需要验证码 |
| 3 | 4  | 发送短信/邮件验证码时需要验证码（防短信轰炸） |

示例：`CaptchaScene = 6`（二进制 110）= 注册 + 发验证码需要图片验证码，登录不需要。

### 前端流程

```typescript
// 获取登录配置
const config = await api.user.getLoginConfig();

// 若需要图片验证码，先获取
let captchaId = '';
let captchaCode = '';
if (config.data?.login?.captcha) {
  const captcha = await api.user.getCaptcha();
  captchaId = captcha.data.captchaId;
  // 将 captcha.data.image (SVG文本) 显示给用户
}

// 登录时附带验证码信息
await api.user.login({ username, password, captchaId, captchaCode });
```

## 8. MFA 集成指南

### 登录流程（含 MFA）

```
POST /Auth/Login
  │
  ├── 正常响应 → code=0, data.accessToken 非空 → 登录成功
  │
  └── MFA 响应 → code=0, data=null, message="mfa_required:{mfaToken}"
        │
        └── 前端显示 6 位验证码输入框
              │
              └── POST /Mfa/Verify { mfaToken, code }
                    ├── code=0 且 data.accessToken 非空 → 登录成功
                    └── code≠0 → 显示错误，重新输入
```

### 用户开启 MFA 流程

```
GET /Mfa/Setup → 获取 totpUri + secret
  │
  ├── 展示二维码（用 totpUri 生成）和手动输入密钥（secret）
  │
  └── 用户用 Authenticator App 扫码后输入 6 位验证码

POST /Mfa/Activate?code={6位码} → 激活成功，返回 10 个备用码
  │
  └── 妥善保存备用码（每个只能用一次，用于忘记手机时紧急登录）
```
