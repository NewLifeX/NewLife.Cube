本文档定义魔方所有前端皮肤登录页的功能需求、接口规范、配置项说明及各皮肤实现细节。

## 2. 后端接口规范

### 2.1 获取站点信息

```
GET /Auth/SiteInfo
```

**响应示例：**

```json
{
  "code": 0,
  "data": {
    "displayName": "魔方WebApi",
    "copyright": "©2002-2026 NewLife Powered by .NET 9.0.0",
    "registration": "粤ICP备16014330号-1",
    "loginTip": "",
    "logo": "",
    "loginLogo": "",
    "loginBg": ""
  }
}
```

**说明：**
- `copyright`：服务端已替换 `{now:yyyy}` 等变量，前端直接显示（Vue 皮肤用 `v-html` 渲染，支持 HTML 链接）
- `loginLogo` / `loginBg`：为空时前端降级到皮肤内置静态资源

### 2.2 获取登录配置

```
GET /Auth/Config
```

**响应示例：**

```json
{
  "code": 0,
  "data": {
    "displayName": "魔方WebApi",
    "logo": "",
    "allowLogin": true,
    "allowRegister": true,
    "enableSms": false,
    "enableMail": false,
    "loginTip": "",
    "providers": [
      {
        "name": "NewLife",
        "logo": "/Content/images/logo/NewLife.png",
        "nickName": "新生命用户中心"
      }
    ]
  }
}
```

**说明：**
- `providers`：仅返回 `Visible=true` 的 OAuth 提供者
- `providers[].logo`：绝对路径，前端直接作为 `<img src>` 使用，无需处理

### 2.3 密码登录

```
POST /Auth/Login
Content-Type: application/json

{
  "username": "admin",
  "password": "123456"
}
```

**响应：**

```json
{
  "code": 0,
  "data": {
    "accessToken": "...",
    "refreshToken": "..."
  }
}
```

### 2.4 发送验证码

```
POST /Auth/SendCode
Content-Type: application/json

{
  "channel": "Sms",     // 手机短信：Sms；邮箱：Mail
  "username": "13800138000",
  "action": "login"
}
```

**响应：**

```json
{
  "code": 0,
  "data": 123456789   // 验证码记录ID
}
```

**注意：** `channel` 值区分大小写，手机为 `Sms`，邮箱为 `Mail`。

### 2.5 验证码登录

```
POST /Auth/LoginByCode
Content-Type: application/json

{
  "username": "13800138000",   // 手机号或邮箱
  "password": "1234",           // 验证码
  "loginCategory": 1            // 1=手机，2=邮箱
}
```

**响应：** 同密码登录。

### 2.6 注册（MVC 版接口，SPA 版前端跳转到注册页）

SPA 皮肤（Vue/React 等）：前端跳转到独立 `/register` 路由页面。  
MVC 皮肤（Metronic8 等）：`POST /Admin/User/Register`，表单提交。

## 3. 配置项说明（CubeSetting）

### 界面配置

| 配置键 | 说明 | 默认值 |
|--------|------|-------|
| `LoginTip` | 登录页提示信息，留空不显示 | 空 |
| `LoginLogo` | 登录页左上角 Logo 图片路径，留空使用皮肤内置 | 空 |
| `LoginBg` | 登录页左侧背景大图路径，留空使用皮肤内置 | 空 |
| `Copyright` | 版权文字，支持 `{now:yyyy}`/`{runtime}`/`{framework}` 变量 | 空（不显示） |
| `Registration` | 备案号，留空不显示 | `沪ICP备10000000号` |

### 用户登录

| 配置键 | 说明 | 默认值 |
|--------|------|-------|
| `AllowLogin` | 允许密码登录 | `true` |
| `AllowRegister` | 允许注册 | `true` |
| `EnableSms` | 启用短信验证码登录 | `false` |
| `EnableMail` | 启用邮箱验证码登录 | `false` |

## 4. 前端实现规范

### 4.1 SPA 皮肤（Vue / ArcoVue / NaiveUI / TDesign / React / Shadcn）

#### 登录页结构

```
/login
  ├── 左侧装饰区
  │     ├── Logo（loginLogo 或皮肤内置）
  │     └── 背景图（loginBg 或皮肤内置）
  └── 右侧表单区
        ├── 系统名称
        ├── Tab 标签页
        │     ├── 密码登录（allowLogin=true 显示）
        │     ├── 手机验证码（enableSms=true 显示）
        │     └── 邮箱验证码（enableMail=true 显示）
        ├── OAuth 第三方登录（providers 非空时显示，分割线 + 图标列表）
        ├── 注册入口（allowRegister=true 显示）
        └── 底部版权信息（copyright / registration）

/register
  └── 注册表单（用户名、邮箱、密码、确认密码）
```

#### OAuth 按钮渲染细节

- 图标：`<img :src="item.logo" />`，`item.logo` 由后端返回绝对路径，无需前端处理
- 文字：优先 `item.nickName`，降级到 `item.name`
- 登录跳转：`window.location.href = '/Sso/Login/' + item.name`（非路由跳转，需全页刷新以完成 OAuth 流）

#### 版权渲染细节

- `copyright` 字段已由服务端替换变量（含 HTML 链接），前端使用 `v-html`（Vue）或 `dangerouslySetInnerHTML`（React）渲染
- `registration` 字段为纯文本，包裹在 `<a href="https://www.beianx.cn/">` 链接中

#### 验证码发送逻辑

1. 点击"发送验证码"按钮，调用 `POST /Auth/SendCode`
2. 成功后开始 60 秒倒计时，按钮禁用并显示剩余秒数
3. 倒计时结束后按钮恢复可点击状态
4. 失败时显示错误信息，按钮保持可用

### 4.2 MVC 皮肤（Metronic8 / Metronic / ElementUI / LayuiAdmin）

- Tab 结构使用 Bootstrap 的 `data-toggle="tab"` 切换
- 验证码发送使用 jQuery AJAX 调用 `/Admin/User/SendVerifyCode`（原有 MVC 端点）
- 注册使用 Bootstrap Tab 切换到本页内的注册面板（`#Register`）
- 版权信息调用 `config.GetCopyright()` 方法渲染

## 5. 数据流图

```
浏览器加载登录页
    │
    ├── GET /Auth/SiteInfo → 展示版权、更新 document.title、获取 loginBg/loginLogo
    │
    └── GET /Auth/Config → 控制 Tab 显示/隐藏、渲染 OAuth 提供者列表

用户点击密码登录
    └── POST /Auth/Login → 成功后跳转首页

用户点击手机验证码登录
    ├── POST /Auth/SendCode { channel: "Sms", username: "手机号" }
    └── POST /Auth/LoginByCode { username: "手机号", password: "验证码", loginCategory: 1 }

用户点击邮箱验证码登录
    ├── POST /Auth/SendCode { channel: "Mail", username: "邮箱" }
    └── POST /Auth/LoginByCode { username: "邮箱", password: "验证码", loginCategory: 2 }

用户点击第三方登录
    └── Location: /Sso/Login/{name}[?r=returnUrl] → OAuth 流程由后端处理

用户点击注册
    └── SPA: router.push('/register')
        MVC: Bootstrap Tab 切换到 #Register 面板
```

## 6. 接口兼容性说明

- `/Auth/*` 系列接口由 `AuthController` 提供，适配 SPA 前端
- `/Admin/User/*` 原有 MVC 接口保留，供 MVC 皮肤和旧版项目使用
- 两套接口共享 `UserService` 业务层，行为一致
