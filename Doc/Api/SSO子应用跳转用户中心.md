# SSO 子应用管理页面跳转用户中心

> 待实现功能。第三代魔方（NewLife.Cube）API 前后端分离版本的 SSO 增强特性。

---

## 需求背景

在典型的魔方 SSO 部署中，一般使用一个纯净的魔方（CubeSSO）作为用户中心，管理所有用户的准入以及角色控制。其它多个子应用引用 NewLife.Cube，关闭密码登录并指向用户中心。

用户在各子系统通过 SSO 登录后，个人信息及角色部门会同步到子应用。因此在子应用里修改个人信息、角色、部门实际没有意义——数据源头在用户中心。

### 问题描述

当前子应用的用户菜单（如"个人信息"链接）仍指向子应用本地页面，用户在子应用中看到的个人信息、角色等管理入口实际无法产生有效修改。用户期望这些链接能够直接跳转到 SSO 用户中心对应的管理页面。

### 期望效果

- 子应用的"个人信息"等用户管理链接，可配置为跳转到 SSO 用户中心对应页面
- 前后端分离版本中，子应用直接使用用户中心的用户管理功能，不在本地维护
- 提供开关配置，允许管理员选择是否启用此跳转行为（非所有场景都需要）

---

## 功能设计

### 配置项

在 `Setting.cs` 中新增配置项（建议）：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `SsoUserCenter` | String | 空 | SSO 用户中心地址，配置后子应用用户管理链接将跳转至此地址 |
| `RedirectUserToSso` | Boolean | false | 是否将用户管理页面（个人信息、修改密码等）重定向到 SSO 用户中心 |

### 影响范围

1. **用户菜单**：顶部导航栏的"个人信息"链接
2. **用户管理页面**：个人信息编辑、密码修改等
3. **前端皮肤**：各皮肤模板中的用户菜单链接需根据配置动态生成

### 实现思路

1. 读取 `SsoUserCenter` 配置，若非空且 `RedirectUserToSso` 为 true，则：
   - 用户菜单中的"个人信息"链接改为 `{SsoUserCenter}/Admin/User/Info`
   - 修改密码链接改为 `{SsoUserCenter}/Admin/User/ChangePassword`
2. API 前后端分离版本中，前端直接调用用户中心的 API 接口
3. 各前端皮肤（ACE、Tabler、LayuiAdmin 等）需适配此配置
