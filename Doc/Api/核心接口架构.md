# 魔方 WebAPI 核心接口架构（前端皮肤对接标准）

> 版本：2026-04-16  
> 适用范围：NewLife.Cube 第三代（WebAPI 主线）

---

## 1. 文档目标

本文档用于统一各前端皮肤（Vue/React/Arco/Naive/TDesign 等）对接 NewLife.Cube 的 API 规范，重点定义：

1. 最小可运行接入所需接口（必要接口）。
2. 可选增强能力接口（高级接口）。
3. 认证、CRUD、OAuth 的统一语义与参数约定。

---

## 2. 必要接口与高级接口

### 2.1 必要接口（最小可运行集合）

> 前端可硬编码站点名称和登录入口，因此 `/Cube/SiteInfo` 与 `/Auth/LoginConfig` 不属于必要接口。

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 认证 | POST | `/Auth/Login` | 密码登录（`loginCategory` 为空表示密码登录） |
| 认证 | POST | `/Auth/Refresh` | 刷新令牌 |
| 认证 | GET | `/Auth/Info` | 当前用户信息 |
| 认证 | POST | `/Auth/Logout` | 登出 |
| 菜单 | GET | `/Cube/MenuTree` | 当前用户可访问菜单树 |
| 实体 | GET | `/{type}` | 列表查询（分页/筛选/排序） |
| 实体 | GET | `/{type}/Detail?id=123` | 详情 |
| 实体 | POST | `/{type}` | 新增 |
| 实体 | PUT | `/{type}` | 编辑 |
| 实体 | DELETE | `/{type}?id=123` | 单条删除 |

### 2.2 高级接口（按需实现）

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 登录配置 | GET | `/Auth/LoginConfig` | 登录方式、OAuth 提供商、开关配置 |
| 站点信息 | GET | `/Cube/SiteInfo` | 标题、Logo、版权等 |
| 安全增强 | GET | `/Auth/Challenge` | Challenge-Response 公钥与 challengeId |
| 验证码 | POST | `/Auth/SendCode` | 发送手机/邮箱验证码 |
| 验证码登录 | POST | `/Auth/LoginByCode` | 验证码登录 |
| 忘记密码 | POST | `/Auth/ResetPassword` | 验证码重置密码 |
| 注册 | POST | `/Auth/Register` | 统一注册 |
| OAuth 注册辅助 | GET | `/Auth/OAuthPendingInfo` | OAuth 回跳预填信息 |
| 元数据 | GET | `/{type}/GetPage` | 页面字段元数据 |
| 导出 | GET | `/{type}/ExportFile` | 数据导出 |
| 导入 | POST | `/{type}/ImportFile` | 数据导入 |
| 上传 | POST | `/{type}/UploadFile` | 附件上传 |
| 图表 | GET | `/{type}/GetChartData` | 图表数据 |
| 批量删除 | DELETE | `/{type}?id=1&id=2&id=3` | 重复参数形式 |

## 3. 最小接入流程

```text
Step 1  POST /Auth/Login
Step 2  GET  /Auth/Info
Step 3  GET  /Cube/MenuTree
Step 4  GET  /{type}
Step 5  POST/PUT/DELETE /{type}
Step 6  accessToken 过期时调用 POST /Auth/Refresh
```

## 4. 认证接口规范（AuthController）

### 4.1 登录（必要）

```http
POST /Auth/Login
Content-Type: application/json
```

请求示例：

```json
{
  "username": "admin",
  "password": "admin",
  "loginCategory": "",
  "challengeId": ""
}
```

字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| `username` | 是 | 用户名 |
| `password` | 是 | 密码明文，或加密密文 |
| `loginCategory` | 否 | 字符串枚举：`""`（密码）、`mobile`、`mail`、`oauth` |
| `challengeId` | 否 | 走 Challenge-Response 时传入 |

### 4.2 验证码登录（高级）

```http
POST /Auth/LoginByCode
Content-Type: application/json
```

请求示例：

```json
{
  "username": "13800138000",
  "password": "123456",
  "loginCategory": "mobile"
}
```

说明：
- `loginCategory` 在验证码登录场景仅允许 `mobile` 或 `mail`。
- `password` 字段承载验证码。

### 4.3 Challenge（高级）

```http
GET /Auth/Challenge
```

响应示例：

```json
{
  "code": 0,
  "data": {
    "challengeId": "a3f2c8e1-4b7d-11ef-9012-0800200c9a66",
    "publicKey": "-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----"
  }
}
```

> 本文档不提供前端加密代码示例。

### 4.4 发送验证码（高级）

```http
POST /Auth/SendCode
Content-Type: application/json
```

请求示例：

```json
{
  "channel": "Sms",
  "username": "13800138000",
  "action": "Login"
}
```

### 4.5 忘记密码（高级）

```http
POST /Auth/ResetPassword
Content-Type: application/json
```

请求示例：

```json
{
  "username": "13800138000",
  "code": "123456",
  "newPassword": "NewPass@2026",
  "confirmPassword": "NewPass@2026"
}
```

### 4.6 统一注册（高级）

```http
POST /Auth/Register
Content-Type: application/json
```

`registerCategory` 使用字符串枚举：

- `""`：用户名密码注册（默认）
- `mobile`：手机验证码注册
- `mail`：邮箱验证码注册
- `oauthbind`：OAuth 回跳后绑定注册

#### 注册示例 1：用户名密码

```json
{
  "registerCategory": "",
  "username": "newuser",
  "email": "newuser@example.com",
  "password": "Strong@2026",
  "confirmPassword": "Strong@2026"
}
```

#### 注册示例 2：手机验证码

```json
{
  "registerCategory": "mobile",
  "mobile": "13800138001",
  "username": "13800138001",
  "code": "123456",
  "password": "Strong@2026",
  "confirmPassword": "Strong@2026"
}
```

#### 注册示例 3：邮箱验证码

```json
{
  "registerCategory": "mail",
  "email": "newuser@example.com",
  "username": "newuser@example.com",
  "code": "123456",
  "password": "Strong@2026",
  "confirmPassword": "Strong@2026"
}
```

#### 注册示例 4：OAuth 绑定注册

```json
{
  "registerCategory": "oauthbind",
  "oauthToken": "6d58fdf6...",
  "username": "newuser",
  "password": "Strong@2026",
  "confirmPassword": "Strong@2026"
}
```

## 5. OAuth 流程（登录与注册分离）

### 5.1 OAuth 已绑定用户：直接登录

```text
Step 1  前端跳转 /Sso/Login/{provider}?r=/admin/index
Step 2  后端重定向到 OAuth 提供商，用户完成授权
Step 3  回跳后端，识别到已绑定用户
Step 4  直接颁发 accessToken/refreshToken
```

### 5.2 OAuth 首次用户：进入注册

```text
Step 1  前端跳转 /Sso/Login/{provider}?r=/auth/register
Step 2  后端重定向到 OAuth 提供商，用户完成授权
Step 3  回跳后端，识别到未绑定用户，返回 oauthToken
Step 4  前端调用 GET /Auth/OAuthPendingInfo?token=xxx 获取预填信息
Step 5  前端提交 POST /Auth/Register（registerCategory=oauthbind）
Step 6  注册成功后颁发 accessToken/refreshToken
```

## 6. 实体 CRUD 规范

### 6.1 列表（必要）

```http
GET /{type}?pageIndex=1&pageSize=20&sort=Id&desc=true&Q=关键字&dtStart=...&dtEnd=...
```

### 6.2 详情（必要）

```http
GET /{type}/Detail?id=123
```

### 6.3 新增（必要）

```http
POST /{type}
```

### 6.4 编辑（必要）

```http
PUT /{type}
```

### 6.5 删除（必要）

```http
DELETE /{type}?id=123
```

### 6.6 批量删除（高级）

```http
DELETE /{type}?id=1&id=2&id=3
```

> 采用重复参数形式，不使用逗号拼接形式。条件删除必须包含至少一个筛选条件。
