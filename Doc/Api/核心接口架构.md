# 魔方 WebAPI 核心接口架构

魔方第三代（NewLife.Cube）采用前后端分离架构，通过统一 RESTful API 为 10 个 SPA 前端皮肤提供数据服务。本文档描述后端接口设计和前端共享包的对应关系。

---

## 1. 总体架构

```
┌─────────────────────────────────────────────────────────────┐
│                     10 SPA 前端皮肤                         │
│  NaiveUI / MUI / Shadcn / ArcoVue / TDesign / Vuetify      │
│  Angular / Svelte / Vue (Element Plus) / React (Ant Design) │
├─────────────────────────────────────────────────────────────┤
│              @cube 共享包（Monorepo workspace）              │
│  api-core / field-mapping / page-utils / auth-logic / page-logic │
├─────────────────────────────────────────────────────────────┤
│                  HTTP / JSON (Authorization Header)          │
├─────────────────────────────────────────────────────────────┤
│               NewLife.Cube WebAPI 控制器                     │
│  AuthController / ReadOnlyEntityController / EntityController│
│  EntityTreeController / CubeController / SsoController       │
├─────────────────────────────────────────────────────────────┤
│           NewLife.XCode 实体层 + UserService 等              │
└─────────────────────────────────────────────────────────────┘
```

### 响应结构

所有 API 返回统一的 `ApiResponse<T>` 格式：

```json
{
  "code": 0,
  "message": "ok",
  "data": { ... },
  "page": { "pageIndex": 1, "pageSize": 20, "totalCount": 150 },
  "stat": { ... }
}
```

| 字段 | 说明 |
|------|------|
| `code` | 状态码，`0` 表示成功，非零为错误码 |
| `message` | 提示消息 |
| `data` | 业务数据 |
| `page` | 分页信息（仅列表接口返回） |
| `stat` | 统计行（仅列表接口可选返回） |

## 2. 认证接口（AuthController）

新版统一认证路径 `/Auth/*`，由 `AuthController` 提供。旧版 `/Admin/User/*` 路径由 `UserController` 保留，用于 SSO 回调和旧版项目。

### 接口速览

| 方法 | 路径 | 说明 | 认证 |
|-----|------|------|------|
| POST | `/Auth/Login` | 密码登录，支持 Challenge-Response 加密密码 | 匿名 |
| POST | `/Auth/SendCode` | 发送手机/邮箱验证码 | 匿名 |
| POST | `/Auth/LoginByCode` | 验证码登录 | 匿名 |
| POST | `/Auth/Refresh` | 刷新令牌 | 匿名 |
| GET | `/Auth/Info` | 获取当前用户信息 | Bearer |
| GET | `/Auth/LoginConfig` | 登录页配置（OAuth 提供商、开关等） | 匿名 |
| POST | `/Auth/Logout` | 登出 | 匿名 |
| GET | `/Auth/Challenge` | 获取 RSA 公钥（Challenge-Response 加密前置） | 匿名 |
| POST | `/Auth/ResetPassword` | 忘记密码——通过验证码重置密码 | 匿名 |

### 2.1 密码登录

```
POST /Auth/Login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin",
  "loginCategory": 0,
  "pkey": ""
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `username` | 是 | 用户名 |
| `password` | 是 | 密码明文，或经 Challenge-Response 加密后的 Base64 密文 |
| `loginCategory` | 否 | 登录方式，默认 `0`（密码登录） |
| `pkey` | 否 | Challenge 挑战密钥 ID，非空时后端用对应私钥解密 `password` |

**LoginCategory 枚举**：

| 值 | 类型 | 说明 |
|---|------|------|
| 0 | Password | 密码登录 |
| 1 | Phone | 手机验证码登录 |
| 2 | Email | 邮箱验证码登录 |
| 3 | OAuth | 第三方登录 |

**响应**：

```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJ...",
    "refreshToken": "Rf...",
    "expireIn": 7200
  }
}
```

> 当 `CubeSetting.AllowPlainPassword = false` 时，若 `pkey` 为空（即未走 Challenge-Response 流程），服务端拒绝登录并返回 `code != 0`。

### 2.2 发送验证码

```
POST /Auth/SendCode
Content-Type: application/json

{
  "channel": "Sms",
  "username": "13800138000",
  "action": "Login"
}
```

| 字段 | 说明 |
|------|------|
| `channel` | 发送渠道：`Sms`（手机短信）或 `Mail`（邮箱） |
| `username` | 手机号或邮箱地址 |
| `action` | 用途：`Login`（登录）、`Register`（注册）、`ResetPassword`（重置密码） |

**响应**：

```json
{
  "code": 0,
  "data": 123456789
}
```

`data` 为验证码记录 ID，仅用于调试追踪，前端无需使用。

### 2.3 验证码登录

```
POST /Auth/LoginByCode
Content-Type: application/json

{
  "username": "13800138000",
  "password": "123456",
  "loginCategory": 1
}
```

`password` 字段填写验证码，`loginCategory` 设为 `1`（Phone）或 `2`（Email）。响应格式同 2.1。

### 2.4 刷新令牌

```
POST /Auth/Refresh
Content-Type: application/json

{
  "refreshToken": "Rf...",
  "userName": "admin"
}
```

**响应**：

```json
{
  "code": 0,
  "data": {
    "token": "eyJ...",
    "refreshToken": "Rf...",
    "expireIn": 7200
  }
}
```

### 2.5 获取当前用户信息

```
GET /Auth/Info
Authorization: Bearer {accessToken}
```

返回用户详细信息，包含权限位图（`permission`）和角色名数组（`roleNames`）。

**响应示例**：

```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "admin",
    "displayName": "管理员",
    "permission": 4294967295,
    "roleNames": ["管理员"]
  }
}
```

### 2.6 获取登录页配置

```
GET /Auth/LoginConfig
```

**响应示例**：

```json
{
  "code": 0,
  "data": {
    "displayName": "魔方WebApi",
    "logo": "",
    "allowLogin": true,
    "allowRegister": true,
    "enableSms": false,
    "enableMail": false,
```

### 2.8 获取 RSA 挑战公钥（Challenge-Response）

```
GET /Auth/Challenge
```

前端在提交密码前先调用此接口获取 RSA 公钥，再用公钥加密密码后提交 `POST /Auth/Login`，实现密码传输不走明文。

**响应示例**：

```json
{
  "code": 0,
  "data": {
    "pkey": "a3f2c8e1-4b7d-11ef-9012-0800200c9a66",
    "publicKey": "-----BEGIN PUBLIC KEY-----\nMIIBIjANBgkq...\n-----END PUBLIC KEY-----"
  }
}
```

| 字段 | 说明 |
|------|------|
| `pkey` | 挑战密钥 ID，提交 `/Auth/Login` 时原样回传 |
| `publicKey` | PKCS#8 SPKI 格式 PEM 公钥，用于 RSA-OAEP/SHA-256 加密 |

**前端加密示例（Web Crypto API）**：

```javascript
async function encryptRSA(password, pemPublicKey) {
  const pem = pemPublicKey.replace(/-----.*?-----|\n/g, '');
  const der = Uint8Array.from(atob(pem), c => c.charCodeAt(0));
  const key = await crypto.subtle.importKey(
    'spki', der.buffer,
    { name: 'RSA-OAEP', hash: 'SHA-256' },
    false, ['encrypt']
  );
  const encrypted = await crypto.subtle.encrypt(
    { name: 'RSA-OAEP' },
    key,
    new TextEncoder().encode(password)
  );
  return btoa(String.fromCharCode(...new Uint8Array(encrypted)));
}

// 调用示例
const { data } = await axios.get('/Auth/Challenge');
const encrypted = await encryptRSA(password, data.data.publicKey);
await axios.post('/Auth/Login', { username, password: encrypted, pkey: data.data.pkey });
```

**安全特性**：
- 每个 `pkey` 有效期 300 秒，使用一次后立即失效，防止重放攻击
- 私钥仅存于服务端内存缓存，不落磁盘
- 后端使用 `RSA-OAEP/SHA-256` 解密，与浏览器 Web Crypto API 加密算法对应

### 2.9 忘记密码——重置密码

```
POST /Auth/ResetPassword
Content-Type: application/json

{
  "username": "13800138000",
  "code": "1234",
  "newPassword": "NewPass@2026",
  "confirmPassword": "NewPass@2026"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `username` | 是 | 手机号或邮箱地址（须与发送验证码时一致） |
| `code` | 是 | 短信/邮件验证码 |
| `newPassword` | 是 | 新密码 |
| `confirmPassword` | 是 | 确认密码，须与 `newPassword` 完全相同 |

**调用前置**：先调用 `POST /Auth/SendCode` 并指定 `action: "ResetPassword"` 发送验证码。

**响应**：

```json
{
  "code": 0,
  "message": "密码重置成功",
  "data": true
}
```

**忘记密码完整流程**：

```
用户输入手机号/邮箱
    └── POST /Auth/SendCode { channel: "Sms"/"Mail", username, action: "ResetPassword" }
            ↓ 用户收到验证码
用户输入验证码 + 新密码 + 确认密码
    └── POST /Auth/ResetPassword { username, code, newPassword, confirmPassword }
            ↓ 成功后跳转到登录页
```

## 3. 数据 CRUD 接口

由 `ReadOnlyEntityController<TEntity>` 和 `EntityController<TEntity, TModel>` 提供。所有路径以实体类型前缀 `{type}` 开头，如 `/Admin/User`、`/Admin/Role`。

### 3.1 列表查询

```
GET /{type}?pageIndex=1&pageSize=20&sort=Id&desc=True&key=value
```

| 参数 | 说明 |
|------|------|
| `pageIndex` | 页码（从 1 开始） |
| `pageSize` | 每页大小 |
| `sort` | 排序字段名 |
| `desc` | 是否降序（`True`/`False`） |
| `dtStart` / `dtEnd` | 时间范围（可选） |
| `Q` | 全文搜索关键词（可选） |
| 其它任意字段名 | 精确筛选 |

**响应**：`ApiResponse<T[]>` + `page` + `stat`

### 3.2 查看详情

```
GET /{type}/Detail?id=123
```

### 3.3 新增

```
POST /{type}
Content-Type: application/json

{ "name": "test", ... }
```

### 3.4 编辑

```
PUT /{type}
Content-Type: application/json

{ "id": 123, "name": "updated", ... }
```

### 3.5 删除

```
DELETE /{type}?id=123
```

### 3.6 批量删除

**按 ID 批量删除**：`id` 参数支持逗号分隔多个主键，与单删接口共用同一路由。

```
DELETE /{type}?id=1,2,3
```

**按条件删除（全部）**：提交与列表查询相同的搜索条件参数，删除所有满足条件的数据。

```
DELETE /{type}?k1=v1&k2=v2&k3=v3
```

> **安全约束**：条件删除时后端要求至少携带一个过滤条件，裸 `DELETE /{type}`（无任何参数）会被拒绝，防止误删全表。

## 4. 字段元数据接口

```
GET /{type}/GetFields?kind=1
```

**FieldKind 枚举**：

| 值 | 类型 | 说明 |
|---|------|------|
| 1 | List | 列表页表头 |
| 2 | Detail | 详情页字段 |
| 3 | Add | 新增表单字段 |
| 4 | Edit | 编辑表单字段 |
| 5 | Search | 搜索栏字段 |

返回 `DataField[]`，每个字段包含：

| 属性 | 说明 |
|------|------|
| `name` | 字段名 |
| `displayName` | 显示名称 |
| `typeName` | .NET 类型（`Int32`/`String`/`Boolean`/`DateTime`） |
| `itemType` | UI 控件提示（`mail`/`mobile`/`image`/`file`/`html` 等） |
| `primaryKey` | 是否主键 |
| `nullable` | 是否允许空 |
| `readOnly` | 是否只读 |
| `dataSource` | 数据字典（`{ "1": "启用", "0": "禁用" }`） |
| `url` | 链接模板（支持 `{Id}` 变量替换） |
| `mapField` | 映射字段名（关联显示） |

前端通过 `@cube/field-mapping` 包的 `resolveWidget()` 函数，将 `typeName` + `itemType` 映射为对应 UI 框架的组件类型。

## 5. 统一导出接口

```
GET /{type}/ExportFile?format=excel
```

**支持格式**：

| format 值 | 输出格式 | Content-Type |
|-----------|---------|-------------|
| `excel` / `xlsx` | Excel（CSV 格式兼容） | `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` |
| `csv` | CSV | `text/csv` |
| `json` | JSON | `application/json` |
| `xml` | XML | `application/xml` |

**导出逻辑**：
- 大数据量（>10 万行）自动启用时间分片导出
- 最大导出行数受 `CubeSetting.MaxExport` 配置控制
- 前端通过 `window.open(url)` 直接下载，不走 AJAX

### 导入接口

```
POST /{type}/ImportFile
Content-Type: multipart/form-data

file: <Excel/CSV 文件>
```

## 6. 文件上传接口

魔方支持两种文件上传模式，可根据场景选用或混合使用。

### 6.1 独立先传模式（推荐）

前端在用户选择文件时立即上传，拿到 `attId` 后在表单保存时回传，后端完成附件与主记录的绑定。

**Step 1：上传文件**

```
POST /{type}/UploadFile?id=0&title=合同A001
Content-Type: multipart/form-data

file: <文件>
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `id` | 否 | 主记录主键。`0` 或缺省表示新增场景；大于零时关联到已有记录 |
| `title` | 否 | 附件标题（主记录显示名）。新增时主记录未保存，可由前端传入预期标题；绑定后后端会用实际 entity.ToString() 覆盖 |
| `file` | 是 | multipart/form-data 中的文件本体 |

**响应**：

```json
{
  "code": 0,
  "data": {
    "attId": 123456789,
    "filePath": "/uploads/Order/20260414/123456789.pdf",
    "contentType": "application/pdf"
  }
}
```

**Step 2：表单提交时携带 attId（新增场景）**

```
POST /{type}?attachmentIds=123456789&attachmentIds=987654321
Content-Type: application/json

{ "name": "合同A001", ... }
```

或编辑场景：

```
PUT /{type}?attachmentIds=123456789
Content-Type: application/json

{ "id": 100, "name": "合同A001", ... }
```

后端 Insert/Update 成功后自动将 `attachmentIds` 对应的附件记录的 `Key` 字段更新为新主记录主键，并补写 `Title` 与 `Url`。

> **attachmentIds** 支持 QueryString（`?attachmentIds=123&attachmentIds=456`）或 multipart 表单两种来源，后端通过 `GetAttachmentIds()` 统一读取。

### 6.2 随表单 multipart 混传模式（兼容路径）

适合简单场景：文件与表单字段打包进同一个 multipart 请求，后端 `SaveFiles()` 自动匹配 `IsAttachment()` 字段并保存附件。

```
POST /{type}
Content-Type: multipart/form-data

name=合同A001&contract_file=<文件>
```

后端识别表单字段名与实体附件字段名匹配（或 `{字段名}_attachment`）时自动处理。

## 9. Cube 公共接口

以下接口均由 `CubeController` 提供，路径前缀 `/Cube/`，无需 Area，全局可用。

### 9.1 站点信息

```
GET /Cube/SiteInfo
```

返回站点名称、Logo、版权等信息，**匿名可访问**，通常在应用初始化或登录页加载时调用。

### 9.2 字典查询

```
GET /Cube/Lookup?codes=enable,sex,role
```

返回多个字典数据，`codes` 为逗号分隔的字典标识。用于下拉框、搜索栏等需要选项列表的场景。

## 11. 控制器继承体系

```
ControllerBaseX                          ← 认证、JSON 包装、Tracer
  ├── AuthController                     ← /Auth/Login 等统一认证
  ├── ReadOnlyEntityController<TEntity>  ← 列表/详情/字段/图表/导出
  │     └── EntityController<TEntity, TModel>  ← 新增/编辑/删除/导入/上传
  │           └── EntityTreeController<TEntity, TModel>  ← 树操作
  ├── CubeController                     ← /Cube/* 站点与页面公共服务（SiteInfo/MenuTree/Lookup/文件/地区等）
  └── SsoController                      ← /Sso/* 单点登录
```

### 路由约定

- **Area 路由**：`/[Area]/[Controller]/[Action]`，如 `/Admin/User/Login`
- **无 Area**：`/[Controller]/[Action]`，如 `/Auth/Login`、`/Cube/Lookup`

## 12. 前端共享包

所有共享包位于仓库根目录 `packages/` 下，通过 pnpm workspace 统一管理。

| 包名 | 说明 | 依赖 |
|------|------|------|
| `@cube/api-core` | HTTP 客户端 + Token 管理 + 5 组 API 工厂函数 | axios |
| `@cube/field-mapping` | DataField → UI 组件类型映射（19 种 WidgetType） | 无 |
| `@cube/page-utils` | 零依赖工具函数（URL 解析、权限检查、菜单查找、导出 URL 构建等） | 无 |
| `@cube/auth-logic` | 认证业务逻辑（登录/登出/用户信息/菜单权限），含 Pinia/Zustand 适配器 | api-core, page-utils |
| `@cube/page-logic` | 列表页面编排逻辑（字段加载/CRUD/分页/导出/图表），含 Pinia/Zustand 适配器 | api-core, field-mapping, page-utils |

### 使用模式

**Vue 系皮肤**（Pinia）：

```typescript
import { createPiniaAuthStore } from '@cube/auth-logic/pinia';
import { usePageLogic } from '@cube/page-logic/pinia';

// 认证 Store
const useAuthStore = createPiniaAuthStore(cubeApi);

// 页面逻辑（在组件中）
const page = usePageLogic(cubeApi, '/Admin/User');
await page.loadFields();
await page.loadData();
```

**React 系皮肤**（Zustand）：

```typescript
import { createZustandAuthStore } from '@cube/auth-logic/zustand';
import { createPageStore } from '@cube/page-logic/zustand';

// 认证 Store
const useAuthStore = createZustandAuthStore(cubeApi);

// 页面 Store
const usePageStore = createPageStore(cubeApi, '/Admin/User');
```
