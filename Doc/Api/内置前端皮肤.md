# 内置前端皮肤

## 概述

第三代魔方（WebAPI 版）提供内置前端皮肤包，用户只需引用 NuGet 包即可获得完整的管理后台前端界面，无需任何前端开发知识。

**可选皮肤包**：

| NuGet 包 | 前端技术栈 | 原始项目 |
|----------|-----------|---------|
| `NewLife.Cube.Vue` | Vue 3 + Element Plus + Vite | [QuickVue](https://github.com/NewLifeX/NewLife.QuickVue) |
| `NewLife.Cube.React` | React 18 + Ant Design 5 + UMI Max | [CubeAntd](https://github.com/NewLifeX/NewLife.CubeAntd) |

## 快速开始

### 1. 安装 NuGet 包

在 WebAPI 项目中引用 `NewLife.Cube` 和任一前端皮肤包：

```xml
<!-- 二选一 -->
<PackageReference Include="NewLife.Cube.Vue" Version="1.0.*" />
<PackageReference Include="NewLife.Cube.React" Version="1.0.*" />
```

### 2. 注册中间件

在 `Program.cs` 中注册前端皮肤：

```csharp
// 注册魔方
services.AddCube();

var app = builder.Build();

// 使用魔方
app.UseCube(app.Environment);

// 使用 Vue 前端皮肤（二选一）
app.UseVue(app.Environment);
// 或使用 React 前端皮肤
// app.UseReact(app.Environment);

app.Run();
```

### 3. 运行

```bash
dotnet run
```

浏览器访问即可看到完整的管理后台界面，包含登录页、动态菜单、列表页、表单页等。

### 后端 API 支持

前端皮肤包依赖以下后端 API（均由 `NewLife.Cube` 提供，无需额外开发）：

```
GET  /{area}/{controller}              # 列表查询（含分页/排序/搜索/统计行）
GET  /{area}/{controller}/Detail?id=   # 查看详情
GET  /{area}/{controller}/GetPage      # 页面元数据（setting + list/addForm/editForm/detail/search）
POST /{area}/{controller}              # 新增
PUT  /{area}/{controller}              # 编辑
DELETE /{area}/{controller}?id=        # 删除
GET  /{area}/{controller}/Export{fmt}  # 导出（Excel/Csv/Json/Xml/Zip）
POST /{area}/{controller}/ImportFile   # 导入文件
POST /{area}/{controller}/DeleteSelect # 批量删除
GET  /{area}/{controller}/GetChartData # 图表数据（ECharts option JSON）
GET  /Admin/Index/GetMenuTree          # 菜单树
GET  /Admin/User/Info                  # 当前用户信息
GET  /Cube/GetLoginConfig              # 登录配置
GET  /Cube/Lookup?codes=               # 字典查询
```

## 工作原理

### 嵌入式静态资源

前端编译产物（HTML/JS/CSS）作为嵌入式资源打包到 .NET 程序集中。运行时通过 `CubeEmbeddedFileProvider` 从程序集中读取并响应 HTTP 请求。

```
用户请求 → StaticFiles 中间件 → CompositeFileProvider
                                  ├── PhysicalFileProvider (wwwroot/, 优先)
                                  └── CubeEmbeddedFileProvider (程序集嵌入)
```

**物理文件优先**：如果 `wwwroot/` 目录下存在同名文件，将优先使用物理文件。这允许在不修改程序集的情况下覆盖个别资源文件。

### 动态页面渲染

前端通过 `GetPage` API 获取页面元数据（页面开关 + 字段集合），根据元数据动态渲染列表和表单页面。新增 Controller 和菜单后，前端自动适配，无需额外开发。

### 本地开发

> 说明：仓库根目录已改为“默认不全量递归编译皮肤”。根目录 `pnpm build` 仅构建 `packages/` 公共模块；如需编译指定皮肤，请使用根目录 `pnpm build:vue`、`pnpm build:react` 等专用脚本，或直接进入对应 `web/` 目录执行 `pnpm build`。

#### Vue 皮肤

```bash
cd NewLife.Cube.Vue/web
pnpm install
pnpm dev          # 启动开发服务器
pnpm build        # 构建 → ../wwwroot/
```

#### React 皮肤

```bash
cd NewLife.Cube.React/web
pnpm install
pnpm dev          # 启动开发服务器
pnpm build        # 构建 → ../wwwroot/（max build）
```

### 根目录快捷命令

```bash
pnpm build          # 仅构建 packages/ 共享包
pnpm build:vue      # 仅构建 Vue 皮肤
pnpm build:react    # 仅构建 React 皮肤
pnpm build:all      # 显式构建全部 workspace 前端
pnpm dev:vue        # 仅启动 Vue 皮肤开发服务器
pnpm dev:react      # 仅启动 React 皮肤开发服务器
pnpm dev:all        # 显式启动全部 workspace 开发任务
```

### 构建流程

前端构建产物输出到 `../wwwroot/` 目录，.NET 项目通过 `EmbeddedResource` 将其嵌入程序集：

```xml
<EmbeddedResource Include="wwwroot\**\*" />
```

修改前端代码后执行 `pnpm build`，再执行 `dotnet build` 即可将最新前端打入程序集。

## 与 MVC 皮肤的区别

| 维度 | MVC 皮肤（AdminLTE/Tabler） | 前端皮肤（Vue/React） |
|------|--------------------------|---------------------|
| 依赖 | NewLife.CubeNC (MVC) | NewLife.Cube (API) |
| 渲染 | 服务端 Razor | 客户端 SPA |
| 包含 | Razor Views + CSS/JS | 编译后的 SPA 产物 |
| 定制 | 覆盖 Razor 视图 | Fork 前端源码 |
| 技术栈 | .NET 开发者即可 | 需要前端开发知识 |
| 功能覆盖 | 100%（基准线） | 90%+（动态列表/表单/CRUD/导入导出/图表） |
| 登录页 | 服务端渲染 | 动态配置（GetLoginConfig） |
| 列表功能 | 搜索/排序/分页/导出/导入/图表 | 搜索/排序/分页/导出/导入/图表/批量操作 |
| 性能 | 每次操作需服务端渲染 | SPA 无刷新，体验更流畅 |

### 自定义图表

后端控制器可通过重写 `OnGetChartData` 方法为列表页提供 ECharts 图表：

```csharp
protected override Object[] OnGetChartData(IEnumerable<MyEntity> data)
{
    // 构建 ECharts option 对象，前端自动渲染
    return [new { title = new { text = "统计" }, series = ... }];
}
```
