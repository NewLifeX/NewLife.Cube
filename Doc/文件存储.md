# 文件存储

魔方提供统一的文件存储服务，支持附件管理、本地存储和分布式文件同步。附件实体（Attachment）记录文件元数据，`SaveFile`/`SaveFiles` 方法处理上传流程，`CubeFileStorage` 实现多节点文件自动同步。

## Attachment 实体

`Attachment` 实体位于 `NewLife.Cube/Entity/附件.cs`，用于记录各系统模块使用的文件，支持 Local/NAS/OSS 等多种存储方式。

### 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `Id` | Int64 | 编号（时间戳类型，用于分布式唯一标识） |
| `Category` | String | 业务分类（如实体类型名：Order、User） |
| `Key` | String | 业务主键（如业务对象 ID） |
| `Title` | String | 标题（用于展示） |
| `FileName` | String | 原始文件名 |
| `Extension` | String | 文件扩展名（.pdf、.jpg 等） |
| `Size` | Int64 | 文件大小（字节） |
| `ContentType` | String | 内容类型（application/pdf 等） |
| `FilePath` | String | 相对路径或 OSS 路径 |
| `Hash` | String | 文件 MD5 哈希值 |
| `Source` | String | 源 URL（用于远程抓取） |
| `Url` | String | 访问 URL |
| `Downloads` | Int32 | 下载次数（累加字段） |
| `LastDownload` | DateTime | 最后下载时间 |

### 审计字段

实体集成了 `UserModule`、`TimeModule`、`IPModule`，自动记录：
- `CreateUserId` / `UpdateUserID`：创建者/更新者 ID
- `CreateTime` / `UpdateTime`：创建/更新时间
- `CreateIP` / `UpdateIP`：创建/更新 IP

### 索引

| 索引 | 字段 | 用途 |
|------|------|------|
| `IX_Attachment_Category_Key` | Category, Key | 按业务分类和业务键查询 |
| `IX_Attachment_FilePath` | FilePath | 按文件路径查询 |
| `IX_Attachment_Extension` | Extension | 按扩展名查询 |

### 查询方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `FindById` | `Attachment FindById(Int64 id)` | 按 ID 查找（单对象缓存） |
| `FindAllByCategory` | `IList<Attachment> FindAllByCategory(String category)` | 按业务分类查找 |
| `FindAllByCategoryAndKey` | `IList<Attachment> FindAllByCategoryAndKey(String category, String key)` | 按分类和业务键查找 |
| `FindAllByExtension` | `IList<Attachment> FindAllByExtension(String extension)` | 按扩展名查找 |
| `GetCategoryList` | `IDictionary<String, String> GetCategoryList()` | 获取分类列表（字段缓存） |

```csharp
// 按分类查找

// 按分类和业务键查找
```

## 文件路径生成规则

`BuildFilePath` 方法按照标准规则生成文件存储路径：

```
{Category}\{yyyyMMdd}\{Id}{ext}
```

例如：`Order\20250115\1234567890.pdf`

```csharp
public String BuildFilePath(String path = null)
{
    var file = FilePath;
    if (FileName.IsNullOrEmpty() && !file.IsNullOrEmpty())
        FileName = Path.GetFileName(file);

    // 自动补充扩展名
    var ext = Extension;
    if (ext.IsNullOrEmpty() && !FileName.IsNullOrEmpty())
        ext = Path.GetExtension(FileName);
    Extension = ext;

    // 构造标准路径
    if (file.IsNullOrEmpty())
    {
        if (Id == 0 || Category.IsNullOrEmpty()) return null;
        var time = UploadTime;
        if (time.Year < 2000) time = DateTime.Today;
        FilePath = file = $"{Category}\\{time:yyyyMMdd}\\{Id}{ext}";
    }
    return file;
}
```

## 实体控制器文件上传

`EntityController<T>` 内置文件上传处理。控制器的 `SaveFiles` 方法自动遍历实体字段，识别附件字段并处理上传。

### SaveFiles 方法

```csharp
protected virtual async Task<IList<String>> SaveFiles(TEntity entity, String uploadPath = null)
{
    var fields = Factory.Fields;
    foreach (var fi in fields)
    {
        var dc = fi.Field;
        // 判断字段是否为附件字段
        if (dc.IsAttachment())
        {
            foreach (var file in Request.Form.Files)
            {
                if (file.Name.EqualIgnoreCase(fi.Name, fi.Name + "_attachment"))
                {
                    // 保存单个文件
                    var att = await SaveFile(entity, file, uploadPath, null);
                    if (att != null)
                    {
                        var url = ViewHelper.GetAttachmentUrl(att);
                        // 使用分号分隔多个文件 URL
                        entity.SetItem(fi.Name, list.Join(";"));
                    }
                }
            }
        }
    }
}
```

### SaveFile 方法

`SaveFile` 处理单个文件的完整上传流程：

```csharp
protected virtual async Task<Attachment> SaveFile(TEntity entity, IFormFile file,
    String uploadPath, String fileName)
{
    if (file == null) throw new ArgumentNullException(nameof(file));
    if (fileName.IsNullOrEmpty()) fileName = file.FileName;

    using var span = DefaultTracer.Instance?.NewSpan(nameof(SaveFile),
        new { name = file.Name, fileName, uploadPath });

    // 1. 构建附件对象
    var id = Factory.Unique != null ? entity[Factory.Unique] : null;
    var att = new Attachment
    {
        Category = typeof(TEntity).Name,   // 实体类型名作为分类
        Key = id + "",                      // 业务主键
        Title = entity + "",               // 实体标题
        FileName = fileName,
        Extension = Path.GetExtension(fileName),
        ContentType = GetContentType(fileName)
    };

    // 2. 保存文件到存储
    var rs = await att.SaveFile(file.OpenReadStream(), uploadPath, fileName);

    // 3. 设置文件路径和大小
    if (rs)
    {
        att.FilePath = att.BuildFilePath();
        att.Size = file.Length;
    }

    // 4. 保存附件记录到数据库
    att.Insert();

    // 5. 审计日志
    var log = LogProvider.Provider.CreateLog(type, "上传", rs,
        $"上传 {file.FileName} ，目录 {uploadPath} ，保存为 {att.FilePath}",
        0, null, UserHost);

    return att;
}
```

### 在实体控制器中使用

```csharp
protected override Int32 OnInsert(Order entity)
{
    // 自动保存上传文件到实体附件字段
    SaveFiles(entity);
    return base.OnInsert(entity);
}
```

## 文件下载接口

`CubeController.File` 提供统一的文件下载接口，路径为 `/cube/file?id={Id}`。

```csharp
[HttpGet]
public async Task<ActionResult> File(String id)
{
    if (id.IsNullOrEmpty()) return NotFound("非法附件编号");

    // 去掉装饰性后缀（如 123.pdf 中的 .pdf）
    var p = id.IndexOf('.');
    if (p > 0) id = id[..p];

    // 查询附件记录
    var att = Attachment.FindById(id.ToLong());
    if (att == null) return NotFound("找不到附件信息");

    // 统计下载次数（异步保存，5秒内合并更新）
    att.Downloads++;
    att.LastDownload = DateTime.Now;
    att.SaveAsync(5_000);

    // 若本地文件不存在，尝试远程抓取
    var filePath = att.GetFilePath();
    if (filePath.IsNullOrEmpty() || !System.IO.File.Exists(filePath))
    {
        var url = att.Source;
        if (url.IsNullOrEmpty()) return NotFound("找不到附件文件");

        var rs = await att.Fetch(url);
        if (!rs) return NotFound("附件远程抓取失败");
        filePath = att.GetFilePath();
    }

    // 返回文件
    if (!filePath.IsNullOrEmpty() && System.IO.File.Exists(filePath))
        return PhysicalFile(filePath, att.ContentType, att.FileName);
    else
        return NotFound("附件文件不存在");
}
```

### 下载 URL 格式

| URL 格式 | 说明 |
|----------|------|
| `/cube/file?id=1234567890` | 标准下载链接 |
| `/cube/file?id=1234567890.pdf` | 带扩展名的友好链接（扩展名被自动忽略） |

### 下载流程

1. 解析附件 ID（去掉装饰性扩展名后缀）
2. 从数据库查询 `Attachment` 记录
3. 累加下载次数，使用 `SaveAsync(5_000)` 异步合并保存
4. 定位本地文件，若不存在则触发远程抓取
5. 返回物理文件响应（带正确的 ContentType 和原始文件名）

## 配置参数

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `UploadPath` | String | `Uploads` | 文件上传根目录（相对路径） |
| `FileStorageProvide` | Boolean | false | 是否启用文件分布式提供（P2P 同步） |
| `FileStorageFetch` | Boolean | false | 是否启用远程文件抓取 |

### appsettings.json 配置示例

```json
{
  "CubeSetting": {
    "UploadPath": "Uploads",
    "FileStorageProvide": true,
    "FileStorageFetch": true
  }
}
```

## 分布式文件存储

`CubeFileStorage` 基于星尘（Stardust）的分布式文件存储框架，实现多节点之间自动文件同步。

### 注册服务

```csharp
services.AddCubeFileStorage("MyApp");
```

### 工作原理

`CubeFileStorage` 继承 `DefaultFileStorage`，以 `Attachment` 表作为文件元数据源：

```csharp
public class CubeFileStorage : DefaultFileStorage
{
    public CubeFileStorage(CubeSetting setting, IServiceProvider serviceProvider,
        ICacheProvider cacheProvider, ITracer tracer, ILog log)
    {
        RootPath = setting.UploadPath;           // 根路径
        DownloadUri = "/cube/file?id={Id}";      // 下载 URL 格式
    }
}
```

### 核心功能

| 功能 | 方法 | 说明 |
|------|------|------|
| 获取文件元数据 | `GetLocalFileMeta` | 从 Attachment 表读取文件信息 |
| 扫描缺失文件 | `GetMissingAttachments` | 遍历 Attachment 表，找出本地不存在的文件 |
| 文件同步 | 自动 | 从其他节点拉取缺失文件到本地 |

### 扫描缺失文件

`GetMissingAttachments` 分页遍历附件表，检查每条记录对应的本地文件是否存在：

```csharp
protected override IEnumerable<IFileInfo> GetMissingAttachments(DateTime startTime)
{
    var exp = new WhereExpression();
    if (startTime.Year > 2000) exp &= Attachment._.CreateTime >= startTime;

    while (true)
    {
        var list = Attachment.FindAll(exp, page);
        if (list == null || list.Count == 0) break;

        foreach (var att in list)
        {
            var filePath = att.GetFilePath(RootPath);
            if (!filePath.IsNullOrEmpty() && !System.IO.File.Exists(filePath))
            {
                yield return new NewFileInfo
                {
                    Id = att.Id,
                    Name = att.FileName,
                    Path = att.FilePath,
                    Hash = att.Hash,
                    Length = att.Size,
                };
            }
        }

        if (list.Count < page.PageSize) break;
        page.PageIndex++;
    }
}
```

### 分布式同步流程

```
节点A上传文件 → 保存到本地磁盘 → 写入 Attachment 表
                                       ↓
节点B启动 → 扫描 Attachment 表 → 发现缺失文件
                                       ↓
节点B ← 从节点A拉取文件 ← 通过 DownloadUri 下载
                ↓
节点B保存本地副本 → 后续请求直接返回本地文件
```

## 管理后台

附件管理在管理后台 **魔方管理 → 附件** 中可查看和管理。

支持的操作：
- **列表查看**：按分类、扩展名筛选附件
- **详情查看**：查看文件元数据、下载次数等
- **文件下载**：直接下载附件文件
- **删除管理**：删除附件记录和文件

## 最佳实践

1. **分类命名**：使用实体类型名作为 `Category`，便于按业务模块管理文件
2. **路径规划**：利用默认的 `{Category}\{yyyyMMdd}\{Id}{ext}` 路径格式，避免目录文件过多
3. **多节点部署**：启用 `FileStorageProvide` 和 `FileStorageFetch` 实现文件自动同步
4. **NAS 共享**：如果使用共享存储（NAS），可不启用分布式同步，直接配置 `UploadPath` 到共享目录
5. **哈希校验**：利用 `Hash` 字段进行文件去重和完整性校验
6. **数据保留**：结合 `DataRetentionService` 定期清理过期附件（参见 [数据保留](数据保留.md)）
- 延迟 10 秒初始化，避免应用启动时资源争抢
