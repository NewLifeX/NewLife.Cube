# 访问规则

魔方提供基于规则的访问控制服务，支持 URL、IP、User-Agent 等多维度匹配，可实现 IP 黑名单、路径拦截和请求限流。

## 核心组件

| 组件 | 说明 |
|------|------|
| `AccessRule` | 访问规则实体，持久化规则配置 |
| `AccessService` | 访问控制服务，在请求管道中执行规则匹配 |

## 规则配置

### AccessRule 实体字段

| 字段 | 说明 |
|------|------|
| `Name` | 规则名称 |
| `Priority` | 优先级，数值越大越先匹配 |
| `Enable` | 是否启用 |
| `Action` | 动作类型 |
| `Url` | URL 匹配模式 |
| `UserAgent` | User-Agent 匹配模式 |
| `IP` | IP 地址匹配模式 |
| `User` | 用户名匹配模式 |

### 动作类型

| 枚举值 | 说明 |
|--------|------|
| `Pass` (1) | 放行，匹配后允许访问 |
| `Block` (2) | 拦截，匹配后拒绝访问 |
| `Limit` (3) | 限流，匹配后按维度限制请求频率 |

### 限流维度

| 枚举值 | 说明 |
|--------|------|
| `IP` (1) | 按 IP 地址限流 |
| `User` (2) | 按用户限流 |

## 匹配规则语法

规则字段支持以下语法：

- **精确匹配**：`/api/login` — 完全匹配
- **通配符**：`/api/*` — 星号匹配任意字符
- **反向匹配**：`!192.168.*` — 感叹号前缀表示"不匹配"
- **多值匹配**：`192.168.1.1,10.0.0.1` — 逗号分隔表示"任一匹配"

## 使用示例

### 拦截爬虫

| 字段 | 值 |
|------|-----|
| Name | 拦截爬虫 |
| Action | Block |
| UserAgent | `*spider*,*bot*,*crawler*` |

### IP 白名单

| 字段 | 值 |
|------|-----|
| Name | 仅允许内网访问 |
| Action | Pass |
| IP | `192.168.*,10.*` |
| Priority | 100（高优先级） |

同时添加一条低优先级拦截规则：

| 字段 | 值 |
|------|-----|
| Name | 拦截外网 |
| Action | Block |
| IP | `*` |
| Priority | 1 |

### API 限流

| 字段 | 值 |
|------|-----|
| Name | 登录接口限流 |
| Action | Limit |
| Url | `/admin/user/login` |
| 限流维度 | IP |

## 执行流程

```
请求到达
  └─ AccessService.Valid(url, ua, ip, user, session)
       ├─ 加载所有启用的规则（缓存）
       ├─ 按优先级和 ID 排序
       ├─ 逐条匹配 URL、UserAgent、IP、User
       │    ├─ 命中 Pass → 放行
       │    ├─ 命中 Block → 拦截
       │    └─ 命中 Limit → 检查会话级限流
       └─ 无命中 → 默认放行
```

## 管理后台

访问规则在管理后台 **系统管理 → 访问规则** 中管理：
- 增删改查规则
- 启用/禁用规则
- 调整优先级
- 查看命中统计
