## 2. 密码登录（Challenge-Response）

### TC-PWD-001 正常登录（Challenge-Response 加密）

| 步骤 | 操作 | 期望结果 |
|------|------|---------|
| 1 | 打开登录页 | 页面正常加载 |
| 2 | 输入正确用户名和密码 | 表单可用 |
| 3 | 点击登录 | 浏览器自动调用 `GET /Auth/Challenge` 获取公钥 |
| 4 | （自动）前端用 RSA-OAEP/SHA-256 加密密码 | Network 中可见 `pkey` 参数和密文密码 |
| 5 | 后端验证通过 | 跳转至首页，无报错 |

**适用皮肤**：全部 10 套

## 3. 忘记密码流程（两步验证码重置）

### TC-FP-001 手机号找回密码（成功路径）

| 步骤 | 操作 | 期望结果 |
|------|------|---------|
| 1 | 点击"忘记密码？" | 进入忘记密码步骤一界面 |
| 2 | 输入已绑定手机号，选择短信渠道 | — |
| 3 | 点击"发送验证码" | 调用 `POST /Auth/SendCode`，显示"发送成功"，60s 倒计时 |
| 4 | 点击"下一步" |  进入步骤二 |
| 5 | 输入正确验证码 + 强密码 + 确认密码 | — |
| 6 | 点击"重置密码" | 调用 `POST /Auth/ResetPassword`，显示成功提示 |
| 7 | 跳转回登录页 | 用新密码登录成功 |

**适用皮肤**：全部 10 套（SPA 跳 `/forgot-password` 路由，MVC 切 Tab/Panel）

## 4. OAuth 第三方登录（returnUrl 传递）

### TC-OAUTH-001 OAuth 登录后跳转到指定 returnUrl

| 步骤 | 操作 | 期望结果 |
|------|------|---------|
| 1 | 访问 `https://localhost:5001/Admin/User?redirect=/User/list` | 被重定向到登录页，URL 含 `?redirect=/User/list` |
| 2 | 点击任意第三方 OAuth 按钮 | 跳转 URL 含 `?r=%2FUser%2Flist` |
| 3 | OAuth 完成授权后 | 跳回 `/User/list` |

**适用皮肤**：全部 10 套

---

### TC-OAUTH-002 OAuth 登录无 returnUrl（默认跳首页）

| 步骤 | 操作 | 期望结果 |
|------|------|---------|
| 1 | 直接访问登录页 | URL 无 `redirect` 参数 |
| 2 | 点击 OAuth 按钮 | OAuth URL 无 `?r=` 参数 |
| 3 | OAuth 完成授权后 | 跳回系统默认首页 `../` |

## 5. 单元测试（XUnit）

测试文件：`XUnitTest/AuthControllerTests.cs`

| 测试方法 | 覆盖点 |
|---------|-------|
| `RsaOaepEncryptDecrypt_RoundTrip` | RSA-OAEP 加解密往返正确性 |
| `RsaOaepEncrypt_NonDeterministic` | 每次加密结果不同（随机填充，防重放） |
| `PemSpkiExportImport_RoundTrip` | PEM/SPKI 公钥导出/导入等价验证 |
| `IsEmail_FormatValidation` (Theory) | 邮箱格式正则校验（5 组正反例） |
| `IsMobile_FormatValidation` (Theory) | 手机号格式正则校验（5 组正反例） |
| `WeakPassword_IsRejectedByRegex` (Theory) | 弱密码被正则过滤 |
| `StrongPassword_Passes` (Theory) | 强密码通过校验 |

运行命令：
```bash
dotnet test XUnitTest/XUnitTest.csproj --filter "FullyQualifiedName~AuthControllerTests" -v normal
```
